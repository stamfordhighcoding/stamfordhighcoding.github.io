Create a new Python App:

Paste the following into your main.py:


import os
        
from groq import Groq
                        
client = Groq(
    api_key=os.environ.get("GROQ_API_KEY"),
)
                        
chat_completion = client.chat.completions.create(
    messages=[
        {
            "role": "user",
            "content": "What does the federal interest rate do to the economy?",
        }
    ],
    model="llama-3.3-70b-versatile",
)
                        
print(chat_completion.choices[0].message.content)

Go to groq.com and create a Free API Key under Developers

Copy the display key, and store it somewhere safe. (like notepad, a google doc, or enter it in your search bar)

Create another tab with 'shell', and type pip install groq

Once all packages are downloaded, change the "content" line of your python code to any question you'd like to ask your AI, and run. Your output will appear in the Console tab.

three.js

three.js is an external library built on top of WebGL for rendering in 3D. WebGL requires some very low-level knowledge, which would take a lot of time to get into. three.js is the most optimized, well-documented library to use for our case. To test if your browser can view WebGL and run three.js, confirm you can move the object below:

Hover over me

To view models, .gltf and .glb formats are the best for importing. Otherwise, OBJ, FBX, STL, Collada, and PLY are also possible to use, though not recommended. The latest version as of writing this is r173, published on December 31.

Some of the most important concepts to grasp are camera rendering, how to use lights, create new and add objects, and request animation frames.

A basic understanding of JavaScript is recommended, though this documentation will do its best to pretend you do not know much about it.

To tinker will all of the following, copy the following code into a new HTML file. Try changing the values that are highlighted here.


    <!DOCTYPE html>
    <html lang="en">
    <head>
      <meta charset="utf-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>Three.js Moving Object</title>
      <script type="importmap">
        {
          "imports": {
            "three": "https://cdn.jsdelivr.net/npm/three@0.173.0/build/three.module.js",
            "three/addons/": "https://cdn.jsdelivr.net/npm/three@0.173.0/examples/jsm/"
          }
        }
      </script>
      <style>
        body { margin: 0; overflow: hidden; }
      </style>
    </head>
    <body>
    
      <script type="module">
        import * as THREE from 'three';
        import { OrbitControls } from 'three/addons/controls/OrbitControls.js';
    
        // Scene setup
        const scene = new THREE.Scene();
        scene.background = new THREE.Color("#222");
    
        // Camera
        const camera = new THREE.PerspectiveCamera(75, window.innerWidth / window.innerHeight, 0.1, 100);
        camera.position.set(4, 5, 9);
        camera.lookAt(0, 0, 0);
    
        // Renderer
        const renderer = new THREE.WebGLRenderer({ antialias: true });
        renderer.setSize(window.innerWidth, window.innerHeight);
        document.body.appendChild(renderer.domElement);
    
        // Lights
        const ambientLight = new THREE.AmbientLight(0xffffff, 0.7);
        scene.add(ambientLight);
    
        const directionalLight = new THREE.DirectionalLight(0xffffff, 1);
        directionalLight.position.set(5, 10, 5);
        scene.add(directionalLight);
    
        // Plane (ground)
        const planeGeometry = new THREE.PlaneGeometry(20, 20);
        const planeMaterial = new THREE.MeshStandardMaterial({ color: 0x555555, side: THREE.DoubleSide });
        const plane = new THREE.Mesh(planeGeometry, planeMaterial);
        plane.rotation.x = -Math.PI / 2;
        scene.add(plane);
    
        // Movable Object (Cube)
        const cubeGeometry = new THREE.BoxGeometry(1, 1, 1);
        const cubeMaterial = new THREE.MeshStandardMaterial({ color: 0xff4444 });
        const cube = new THREE.Mesh(cubeGeometry, cubeMaterial);
        cube.position.y = 0.5;
        scene.add(cube);
    
        // Controls
        const controls = new OrbitControls(camera, renderer.domElement);
        controls.enableDamping = true;
    
        // Movement variables
        const moveSpeed = 0.1;
        const keys = { ArrowUp: false, ArrowDown: false, ArrowLeft: false, ArrowRight: false, KeyW: false, KeyA: false, KeyS: false, KeyD: false };
    
        window.addEventListener("keydown", (event) => {
          if (keys.hasOwnProperty(event.code)) keys[event.code] = true;
        });
    
        window.addEventListener("keyup", (event) => {
          if (keys.hasOwnProperty(event.code)) keys[event.code] = false;
        });
    
        // Animation loop
        function animate() {
          requestAnimationFrame(animate);
    
          // Move the cube based on key inputs
          if (keys.ArrowUp || keys.KeyW) cube.position.z -= moveSpeed;
          if (keys.ArrowDown || keys.KeyS) cube.position.z += moveSpeed;
          if (keys.ArrowLeft || keys.KeyA) cube.position.x -= moveSpeed;
          if (keys.ArrowRight || keys.KeyD) cube.position.x += moveSpeed;
    
          controls.update();
          renderer.render(scene, camera);
        }
        animate();
    
        // Resize handling
        window.addEventListener("resize", () => {
          camera.aspect = window.innerWidth / window.innerHeight;
          camera.updateProjectionMatrix();
          renderer.setSize(window.innerWidth, window.innerHeight);
        });
    
      </script>
    
    </body>
    </html>

npm (Node Package Manager) Overview

npm is the default package manager for Node.js, used to install, share, and manage dependencies in JavaScript (or TypeScript) projects.

npm is bundled with Node.js. Make sure you have the latest LTS version from nodejs.org

Verify your installation with node -v and npm -v


Initializing a project - use "npm init" or "npm init -y"

The generated package.json file should include a structure that may be similar to the following:


{
  "name": "my-app",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {},
  "dependencies": {},
  "devDependencies": {}
}

Packages are placed in node_modules/ and their versions recorded in package.json.


Common Comands

Install all dependencies:


npm install

Update Packages:


npm update

Uninstall a Package:


npm uninstall [package name]

List installed packages:


npm list --depth=0

Audit for vulnerabilities


npm audit
npm audit fix

Scripts

In package.json under "scripts"


"scripts": {
  "start": "node server.js",
  "test": "jest", //example
  "build": "webpack --config webpack.prod.js" //example
}

Note that comments are not permitted in json files.


To run, use


npm run build
npm test

Tips and Best Practices

  • Commit your package.json and package-lock.json; ignore node_modules/.
  • Use npx to run binaries without global install.
  • Pin critical dependencies by using exact versions.



Git is a distributed version control system for tracking changes in source code during software development. It lets you record snapshots of your project, collaborate with others, and safely experiment without losing history. In other words, if you want an advanced CTRL+Z with cross-device features, and enterprise-level version control, git is one of the best solutions.

A couple of cases we will be using git is to push code to Github, clone a Github Repository, and keep version control.

Gitlab and Bitbucket are a couple of alternatives to Github that enterprises use.

  • Windows (and others): Download and run the installer from git-scm.com/downloads.
  • macOS (terminal): brew install git (if you have Homebrew)
  • Linux (Debian/Ubuntu): sudo apt-get update && sudo apt-get install git

After installing, configure your identity:


git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Starting a Repository


cd path/to/project
git init

Clone an existing repo:


git clone https://github.com/username/repo.git

Basic Workflow

Check Status


git status

Stage Changes


git add file1.txt          # single file
git add .                  # all changes

Commit Staged Changes


git commit -m "Brief, descriptive message"

View history:


git log --oneline --graph

Remotes & Collaboration

  • Add a remote: git remote add origin https://github.com/username/repo.git
  • First push for upstream: git push -u origin main
  • subsequent pushes git push
  • Fetch & integrate changes git pull origin main

Undoing Changes

  • Unstage a file: git reset HEAD file.txt
  • Discard local changes: git checkout -- file.txt
  • Revert a commit git revert <commit-hash></commit-hash>
  • Reset to a previous commit (destructive): git reset --hard <commit-hash></commit-hash>

Best Practices

  • Commit often with clear messages.
  • Use branches for features/fixes.
  • Pull before you push to minimize conflicts.
  • Review your work (git diff) before committing.
